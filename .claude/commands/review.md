# コードレビュー

以下の観点でプロジェクト全体をレビューしてください:

## チェック項目

1. **型安全性**: `any` が使われていないか、型定義が適切か
2. **エラーハンドリング**: try-catch が適切に配置されているか、ユーザーにエラーが通知されるか
3. **CSV整合性**: CSV出力が src/types/index.ts の CSV_COLUMNS 定義と一致しているか
4. **プラットフォーム分岐**: iOS/Android の違いが適切に処理されているか (Platform.OS)
5. **メモリリーク**: setInterval, EventListener が適切にクリーンアップされているか
6. **コーディング規約**: CLAUDE.md の規約に従っているか

## 出力形式

問題を発見したら以下の形式で報告:
- ファイルパス:行番号
- 問題の種類 (型安全性 / エラー / バグ / 規約違反)
- 問題の説明
- 修正案
